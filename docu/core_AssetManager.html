core.AssetManager

loads and internally caches assets

properties

  • imgs:Object, cache for Image objects
  • snds:Object, cache for Sound objects (not implemented yet)
  • txts:Object, cache for Strings loaded from text-formatted files
  • pendingTexts:Array, internal buffer/queue of XHRs for text assets
  • assetCounter:Number, total assets queued
  • assetsLoaded:Number, total assets that actually loaded
  • queueCompete:Boolean, only true if all assets have been loaded
  • assetsPath:String, the base URL to load assets from

methods

  • onAssetLoaded():void, callback function for when an asset loaded, increments assetsLoaded
  • onXMLHTTPLoaded(id,pendingXMLHTTP,text):void, callback function for when an XHR has successfully loaded, adds the response text to the txt cache with an id of id, and removes the completed XHR from the XHR queue array
  • loadAsset(type,path,id):void, attempts to load an asset of type:type, from path:path, and caches it as id:id, only supports types "image" and "txt"
  • tick():void, update/tick method, updates queueComplete
  • requestAsset(type,ID):Image/String, attempts to fetch an asset from one of the internal caches, returns undefined by default.
