core.InputManager

handles keyboard and mouse events

properties

  • parent:Object (usually Game),
  • timing:Number, cooldown for keys/clicks, defaults to 30, measured in ticks
  • keyStates:Object, holds key states
  • mouseState:Object, holds mouse state (x,y,isDown)

methods

  • InputManager(parent):InputManager, constructor, creates event listeners and sets parent propety to parent
  • checkMouseCollision(e):object{result:Boolean}, checks if the given object is colliding with the mouse pointer
  • checkKeyDown(keyID,[keyName]):Boolean, checks if a key is down
  • checkKeyPress(keyID,[keyName]):Boolean, checks if a key was pressed (only works once if key is held, key must be released and pressed again to work again, otherwise, same as checkKeyDown), BUGGY!
  • tick():void, update/tick method, updates keystate timers
